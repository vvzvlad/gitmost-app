# Docmost

Нативное macOS-приложение, которое показывает веб-интерфейс
[Docmost](https://docmost.com/) для нескольких ваших серверов. Каждый сервер —
отдельная вкладка в одном окне: переключение вкладок переключает серверы, а
сессии (авторизация) сохраняются между запусками.

## Требования

- macOS 14.0 или новее
- Тулчейн Swift (Xcode 15+ или Command Line Tools)

## Сборка и запуск

Основной путь — через `make`:

```sh
make build
make run
```

`make build` собирает релизный бинарник через Swift Package Manager и упаковывает
его в `Docmost.app` с корректным `Info.plist` и иконкой приложения, после чего
подписывает бандл ad-hoc подписью. `make run` собирает приложение и запускает его.

Те же шаги можно вызвать напрямую:

```sh
swift build       # отладочная сборка SwiftPM, без .app
make build        # полная релизная сборка и упаковка в Docmost.app
open Docmost.app  # запуск собранного приложения
```

## Добавление серверов

Откройте «Servers…» в панели вкладок (или нажмите Cmd+N — «Add Server…»),
введите название и адрес сервера. Если схема не указана, автоматически
подставляется `https://`. Список серверов хранится локально.

## Особенности

- Постоянные сессии по доменам: cookie и авторизация сохраняются между запусками
  через постоянное хранилище WebKit, изолированное по доменам.
- Внешние ссылки (клик по ссылке на другой домен) открываются в браузере по
  умолчанию. Редиректы и SSO/OAuth-переходы остаются внутри приложения.
- Файлы, которые нельзя показать внутри, скачиваются в `~/Downloads` и
  показываются в Finder.
- Для загрузки файлов используется нативный диалог выбора (NSOpenPanel).
- Масштаб страницы — ⌘+ / ⌘− / ⌘0; уровень масштаба сохраняется.

## Встроенный JS/CSS

Приложение поставляется со встроенными JavaScript и CSS, которые скомпилированы
в программу и внедряются во вкладку каждого сервера. Внешнего каталога или
редактируемого файла нет. Сейчас встроенный скрипт скрывает недоступный
платный интерфейс «Resolved»-комментариев (вкладку «Resolved» в панели
комментариев и неактивный пункт контекстного меню «Resolve comment»).

Чтобы изменить внедряемый JS/CSS, отредактируйте встроенные константы `js`/`css`
в `Sources/DocmostCore/UserScripts.swift` и пересоберите приложение.

## Разработка

Все рутинные действия — через `make` (см. `make help`):

| Команда      | Назначение                                            |
|--------------|-------------------------------------------------------|
| `make`       | Показать список целей (то же, что `make help`)        |
| `make build` | Релизная сборка `Docmost.app` (компиляция+упаковка)   |
| `make run`   | Собрать и запустить `Docmost.app`                     |
| `make test`  | Прогнать юнит-тесты (`swift test`)                    |
| `make debug` | Отладочная сборка SwiftPM (без `.app`)                |
| `make icon`  | Перегенерировать иконку приложения (`Resources/AppIcon.icns`) |
| `make clean` | Удалить артефакты сборки                              |

Тесты лежат в `Tests/DocmostCoreTests` и запускаются через `make test`. Они
покрывают чистую логику из `Sources/DocmostCore` (модели и хранилище серверов).

## Конфигурация

Приложению не нужны переменные окружения или секреты: серверы настраиваются в
интерфейсе и хранятся локально в `UserDefaults`.
